  • HTML TAG

    • HTML 태그들을 무자비하게 정리

    *꺾쇠는 . 으로 대체

    **대부분 작성 순서 및 중요도 순서대로 작성


    1. 기본 틀

      • .!DOCTYPE HTML / .HTML

        • HTML의 원활한 구동을 위해 꼭! 맨 위에 적고 시작

      • .head

        • 페이지의 가장 중요한 정보를 나타내는 부분.

      • .title

        • 페이지의 이름(?)을 바꿔줌. 검색에서 우선정보가 됨

      • .meta charset="UTF-8"

        • 페이지의 버전(형식). 이걸 안 적으면 한글이 안나올수도 있음. 현재는 대부분 UTF-8로 되니, 저대로 적자


    2. 단락/리스트

      • .h1~6

        • 제목을 표현하는 태그. 각 숫자별로 크기가 다르며, 일반 p태그에 이미지 조정보다 강력한 영향이 있으니 중요할 때 꼭 활용할것

      • .p

        • 페이지의 단락을 결정하는 태그. 웬만하면 이걸로 쓴다

      • .br

        • 엔터

      • .ol, ul

        • 번호가 있는/없는 리스트 작성의 부모.

      • .li

        • ol, ul의 자식 두 태그안에 이것들을 적는 만큼 리스트작성이 된다.


    3. 링크/삽입

      • .a href=""

        • 하이퍼링크 기능. 폴더에서 이미지 업로드 시, 꼭 경로를 정확히 할 것!

        ex) 폴더/이미지.확장자

        target="_blank" 를 붙인다면, 새 창에서 열기 /

      • .a id=""

        • a id="여기" 와 a href="#여기" 를 통해 해당페이지 바로가기 가능

        사실 뭐하는 놈인지 잘 모른다.

      • .img src="이미지 링크 or 사이트 폴더 내에 사진명"

        • 이미지 삽입. Width="" ->이미지 크기조정

        alt="" -> 이미지의 대체텍스트. 설명문이 되어 여러모로 검색에 용이해진다

      • .viedo src="이미지 링크 or 사이트 폴더 내에 영상명"

        • 동영상 삽입. 얘는 뒤에 /로 닫아줘야함. 꼭!

        Width="", heihgt="" ->동영상 크기조정

        controls -> 영상의 속도 등 조정가능한 옵션

      • .audio src="파일 링크 or 사이트 폴더 내에 파일명" controls type="파일형식" /

        • 음악 삽입. + autoplay controls 오토플레이도 가능하다.

      • object src="파일" / embed="파일" /

        • 음악이나 영상 / 움짤이나 링크 등 삽입. 언제쓸까?


    4. 표 작성

      • table

        • 표 전체를 감싸는 큰 틀

      • td

        • 행의 가로 작성. colspan 으로 가로 병합

      • tr

        • 열의 세로 작성. rowspan으로 세로 병합

      • .thead / .tbody / .tfoot

        • 각 표의 맨 위, 몸통, 꼬리를 담당. 어디에 쓰여도 표 위치는 고정이다.


    5. FORM

      • .form action="보낼주소" method="get/post"

        • 텍스트, 체크 등 입력 양식을 보내기 위한 틀. action을 통해 보낼 위치를 지정

      • .input type="text/password" name="지정정보" value="초기 텍스트값"

        • 텍스트를 적는 기본적인 칸. name보낼 정보의 이름 지정, value로 작성 전 기본값 지정

      • .input type="submit" value="변경이름"

        • form 내에 작성한 정보를 제출을 위한 버튼. value값으로 버튼의 이름설정가능

      • .textarea/ cols="가로길이" rows="tpfhrlfdl"

        • 내용을 적는 것 같은 큰 상자

      • .select name="이름지정"/

        • 선택지의 시작.

      • .option value="보낼 정보값"

        • 선택지 내부 내용들. value로 지정한 정보값이 지정되니, 웬만하면 영어로 할 것

      • input type="radio" name="이름,동일시 중복x" value="보내는값" checked-처음지정된것

        • 동그란 체크박스. 각 라디오를 같은 name으로 지정시, 중복선택 X

      • input type="checkbox" name="이름,동일시 중복O" value="보내는값" checked-처음지정된것

        • 네모난 체크박스. 각 체크박스를 같은 name으로 지정해도, 중복선택 O

      • label for="내용" / id="내용"

        • 체크박스, 텍스트 등 같은 내용을 작성시 id로 지정해 둔 값을 레이블로 묶어, 지정범위 확대 사이트 정확성에 기여

      • input type="hide" name="이름" value="전송값"

        • 유저에게 보이지 않지만, 정보전송이 필요한 부분에 사용

      • input type="file" / form action="" + method="post" enctype="multiple/form-date"

        • 파일 전송용. 전송받는 서버에 대해 이해가 낮으니, 참고만 하자.

      • input type="number" id="지정값" name="이름" step="각 범위별 차이"

        • 숫자만을 체크하는 타입. 위/아래로 조절이 가능하며, 검색 또한 가능하다.

      • input type="range" name="이름" id="지정값" stpe="항목별 눈금?" max="" min=""

        • 범위 지정 막대. 좌/우로 움직여 항목의 숫자를 결정하며, 정확한 숫자는 보이지 않음.

      • 데이터리스트

      • 1. input list="B" id="A" name="A"

      • 2. datalist id="B"<./option>

      • 3. option value="지정할 이름, 이외 텍스트 없음."

        • 검색이 가능한 셀렉트 리스트. 꼭 input list와 datalist가 같고, Id와 name이 같게 설정할것

        옵션에서는 다른 텍스트 지정이나 박스 없이, 옵션값에 벨류 넣어서 마무리., 끝에 /값은 꼭 넣어주자


    6. > 구성요소 지정 (레이아웃)

      • div id="지정값" +그외 css요소

        • 내부 구성요소와 레이아웃을 지정. css에서 레이아웃 사이즈를 배우자.

      • header

        • 말 그대로 제목부분

      • nav

        • 목차부분을 지정하는 섹션태그

      • main

        • 말 그대로 메인. 상위 개념은 아니기에, 다른 article 등 구성요소 태그의 상 하위에 다 쓰일수 있다.

      • footer

        • 제일 아래. 주석이나 연락처 등 정보 기입란

      • section id="?"

        • 파트를 구분짓는 대표적 레이아웃. id로 활용가능한듯

      • article

        • 기사나 글의 내용 부분.

      • aside

        • 본문에 관련되지만 맥락에는 관련되지 않는 부가요소들.

      • figure/figcaption

        • 이미지 삽입부분 / 삽입부분의 설명부분. (상, 하위개념)
